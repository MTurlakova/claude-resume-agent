# resume-tailor

## Purpose
Tailor a resume to a specific job description through a guided conversation, then export a formatted .docx file.

## Inputs
- job_description (string): Full job description text, pasted into the chat — or a URL to the job posting.
- source_resume (string): Selected from saved resume files in the project folder. Do not ask the user to paste a resume.
- company (string): Target company name.
- role (string): Target job title.

## Workflow
1. List the available resumes from the `resumes/` folder (base variants) and `resumes/tailored/` folder (previously tailored versions) by reading the directory. Ask the user which to use and ask for the job description (pasted text or URL). If a URL is provided, fetch it with WebFetch and extract the job description text. Note: LinkedIn URLs often block scraping — if a fetch fails, ask the user to paste the text instead. Read the selected resume file automatically — do not ask the user to paste it.
2. Analyze the fit between the resume and the job description. Identify gaps and areas to reframe.
3. Generate 3-5 targeted clarifying questions based on the specific gaps identified — not a generic checklist.
4. Collect answers. Also ask: "Anything else you'd like to add or emphasize?"
5. Produce a tailored resume draft using all gathered context. Rules:
   - Do not fabricate experience. Only use what is in the resume and the candidate's answers.
   - Reframe existing experience to highlight skills relevant to this role.
   - Use language from the job description where it accurately reflects the candidate's experience.
   - Keep bullet points concise and impact-focused.
   - Preserve the candidate's voice.
6. Show the draft and ask if the user wants any changes. Iterate until satisfied.
7. Write the finalized resume content to `resume_data.py` in the project folder.
8. Save the finalized resume as plain text to `resumes/tailored/{Company}_{Role}.txt` for future reuse.
9. Ask: "Ready to generate the .docx?" and if the user confirms, run `python generate_resume.py` directly using Bash.

## Outputs
- Updated `resume_data.py` with the tailored resume content.
- `{output_folder}\{First Last} Resume - {Company} {Role}.docx` generated by `generate_resume.py`.

## Output format defaults
- Font: Garamond 11pt
- Margins: 0.75 inches
- Target length: 2 pages
- Output folder: set via `OUTPUT_FOLDER` in `resume_data.py`

## Notes
- `generate_resume.py` reads from `resume_data.py` and handles all .docx formatting.
- The Anthropic API is not used. Tailoring happens through conversation in Claude Code.
- The user may have multiple source resumes in `resumes/` — always ask which to use before starting.
